# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A Teardown mod adding a gasoline flamethrower tool: adjustable nozzle velocity (via a physical knob, arrow keys by default), an ammo system, and a custom fire simulation that spawns fires along the flame trajectory with `SpawnFire()` rather than relying on the engine's built-in flame behavior. Published on Steam Workshop (ID in `mod/id.txt`).

## Development

Pure Lua, no build/test/lint toolchain. The mod content lives in `mod/` — to iterate, copy or symlink `mod/` into Teardown's mods folder and reload in-game.

- `teardown.d.lua` provides Teardown API type stubs for IDE autocomplete (generated by the sibling `teardown-api-stub-generator` repo).
- Debug mode: toggle **Enable debug overlay** in the mod's options menu (persisted at `savegame.mod.features.debug.enabled`, read by `setup()` on (re)load). Adds raycast visualization and property watches, and swaps in the debug voxel model. Off by default; leave it off in releases.

## Architecture

`mod/main.lua` is the entry point. Scripts are composed with Teardown's `#include` preprocessor directives (not `require()`); modules are defined at file scope. Lifecycle: a single `setup()` re-establishes engine bindings (tool registration, sound loading, registry-read config) and runs from `init()` plus lazily from a one-shot `tick()`/`update()` guard, so it re-runs after a quickload; `init()` additionally applies fresh-start state (full ammo, fire-limit override) that must not be re-applied on load. `update()` (tool active + usable) drives flame spawning and ammo consumption. See `../docs/reference/save-load.md` and `script-lifecycle.md` for the why.

Singletons (`Flamethrower`, `Nozzle`, `Knob`, `SoundManager`, `Debug`, `Engine`) are global tables with methods, some using metatables. Transient `Flame` entities are deliberately **plain data** (`Flame.new` returns a metatable-free table; `Flame.tick`/`Flame.update` are file-scope functions) so they survive quicksave/quickload without crashing — see `../docs/reference/save-load.md`. Flow: holding `usetool` → `Flamethrower:update()` (`mod/scripts/flamethrower.lua`) → `Nozzle:throwFlames()` (`mod/scripts/nozzle.lua`) raycasts and creates `Flame` entities (`mod/scripts/entities/flame.lua`), which each frame advance and spawn fires along their path. `Knob` (`mod/scripts/knob.lua`) adjusts flame velocity (5–25 m/s). `SoundManager` handles start/loop/end sounds. `mod/options.lua` is a separate script Teardown loads for the settings UI (keybind remapping, fire-limit override, inventory slot, debug overlay toggle).

Settings persist via Teardown registry keys under `savegame.mod.features.*`; defaults are only written when the key is unset.

## Gotchas

- Voxel shapes are accessed by hardcoded index (`shapes[2]` = knob, `shapes[3]` = fire starter, `shapes[4]` = nozzle). Editing `mod/vox/Flamethrower.vox` can silently break these bindings.
- The fire-limit option mutates global game state (`game.fire.maxcount`).
- Voxel size constant (0.05) lives in `mod/scripts/lib/engine.lua` and is used throughout transform math.
